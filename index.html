ヒアリング結果のまとめ

現行業務はLINE・Excel(5種)・Dropbox・FAX・電話の組み合わせで運用されており、データの断絶と手作業依存が随所に存在します。
特に「入金繰越」「日程調整漏れ」「事務属人化」は業務品質と売上に直結する課題であり、Salesforce導入で仕組みとして解決できる領域です。
MTG#6で萩本氏より「外部視点からの業務フロー見直し提案を歓迎する」との明言をいただいており、単なるExcel置き換えではなく業務フローの最適化も含めた設計を進めます。

日常業務の変化イメージ

BEFORE（現在）
  • 毎朝 銀行口座を目視確認 → LINEで3段階報告
  • 契約書 FAX受領 → Excel手入力 → 別Excelに転記
  • 日程調整 「電話したっけ？」を記憶で管理
  • セカンド 実施有無はLINE遡りで確認
  • 月末 繰越が判明してから慌てて対応
  • 経営判断 各営業所が口頭で数字報告
AFTER（Salesforce導入後）
  • 毎朝 「事務の朝画面」で未入金リストが赤黄緑で一覧
  • 契約書 スキャン → OCR自動読取 → 確認1クリック
  • 日程調整 未対応リストが自動生成・一目瞭然
  • セカンド 実施状況一覧 / 18時以降は自動ルーティング
  • 月初から 繰越リスク一覧でリアルタイム把握
  • 経営判断 全営業所KPIがダッシュボードでリアルタイム比較

主要リスクと対策

データ整備レベルの課題

「現在のデータ整備レベルはかなり整っていない状態」（千葉 MTG#9）。移行前のクレンジングが必要。

対策: 4月W2でデータ棚卸し。移行対象をPhase 1に必要な範囲に絞りスモールスタート
営業のSalesforce定着リスク

LINE通知に慣れてSalesforceを見なくなるとライナー解約リスク（Murakami氏 MTG#9）。

対策: Phase 2でLINE通知連携を実装し、LINEを起点にSalesforceへ誘導する導線設計
口頭慣習フィールドの移行

歩合計算用番号列（丸1〜3）等、Excel外の口頭ルールに依存するフィールドが複数。

対策: 4月W1で岸氏と全フィールド定義を文書化。不明点は鳥飼氏にも確認

進め方の3つの原則

1. 現場が毎日使う画面から作る — Phase 1で「事務の朝画面」を最速で届け、早期に効果を実感してもらいます。

2. 事務が入力し、営業に渡す — 営業にSalesforce直接入力を強制せず、事務が登録→営業がスマホで確認するオペレーションで定着率を最大化。

3. 小さく出して、速く回す — 各フェーズ末にリリース→検証→改善の1週間サイクルを回し、現場の声を即座に反映します。
